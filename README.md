# AzureNamingBrowserAddin (Chrome Extension)
Browser Addin to generate names for the Azure WebGUI

A powerful browser add-in that generates consistent Azure resource names according to CAF best practices directly in the Azure portal – including hundreds of resource types, custom patterns, and automatic field detection.

Anyone who works a lot in the Azure portal knows the problem:
Every resource needs a consistent, clean name – ideally according to CAF best practices.
After the 20th resource, however, this quickly becomes tedious or error-prone.

To make my life easier, I have developed a small but extremely helpful Chrome plugin:

 <h2> ##Azure Naming Browser Addin  </h2>

Supports official Azure CAF abbreviations
incl. extended list and separate "Diverse" submenu for special resources

Custom naming patterns per resource type
Example: vnet-{Env}-{Location}-{Application}-{Rand4}

Fields: Environment, Location, Application → modern Naming-Standards

Usable directly in the Azure portal

Right-click → Azure Naming → select resource type

Button "AZ Name" in the portal → same selection incl. Divers

Auto-fill in the correct field (no interference with global search)

Extremely fast workflow for architects, DevOps, consultants & admins
